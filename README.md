# SOC Detection Rules

Набор detection rules для SIEM систем. Пишу правила на основе разбора реальных атак — HTB машины, CVE-исследования, публичные отчёты по инцидентам.

## Форматы

Каждое правило стараюсь делать в трёх форматах:

- **Sigma** — универсальный, конвертируется в большинство SIEM
- **MaxPatrol SIEM** — Positive Technologies, распространён в РФ
- **R-Vision SIEM** — тоже встречается часто

## Структура

```
├── sigma-rules/
├── maxpatrol-queries/
├── rvision-queries/
├── research/        # разборы атак, на основе которых написаны правила
├── docs/
└── templates/       # шаблоны для новых правил
```

## Методология

Перед тем как писать правило — разбираю атаку: что именно происходит в системе, какие артефакты остаются в логах, какие из них уникальны для этой техники а не для легитимной активности. Потом тестирую на ложные срабатывания.

Стараюсь привязывать каждое правило к конкретной технике MITRE ATT&CK — удобно для приоритизации и для объяснения аналитикам что именно детектируется.

## Пример

`research/` содержит разбор HTB Giveback — там CVE-2024-5932 (Give plugin RCE). На основе разбора написал правило для детектирования характерных паттернов в веб-логах.
